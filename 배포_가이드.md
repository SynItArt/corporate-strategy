## 🚀 배포 방법

### 1단계: StrategyAbled.html 복사
```bash
# 원본 파일에서 배포용 폴더로 복사
copy "HTML\법인대표님\StrategyAbled.html" "배포용\StrategyAbled.html"
```

또는 수동으로:
- `HTML/법인대표님/StrategyAbled.html` 파일을
- `배포용/StrategyAbled.html`로 복사

### 2단계: 프로필 사진 복사
```bash
# 원본 파일에서 배포용 폴더로 복사
copy "HTML\법인대표님\신대식 사진.png" "배포용\신대식 사진.png"
```

또는 수동으로:
- `HTML/법인대표님/신대식 사진.png` 파일을
- `배포용/신대식 사진.png`로 복사

### 3단계: GitHub 저장소 생성
1. GitHub에서 새 저장소 생성
2. 저장소 이름: `corporate-strategy` (또는 원하는 이름)
3. Public 또는 Private 선택

### 4단계: 파일 업로드
방법 1: GitHub 웹 인터페이스
1. 저장소 페이지에서 "Add file" → "Upload files"
2. `배포용` 폴더의 모든 파일을 드래그 앤 드롭
3. "Commit changes" 클릭

방법 2: Git 명령어
```bash
# 저장소 클론
git clone https://github.com/사용자명/corporate-strategy.git
cd corporate-strategy

# 배포용 폴더의 모든 파일을 루트로 복사
# (Windows)
xcopy /E /I 배포용\* .

# (Mac/Linux)
cp -r 배포용/* .

# 커밋 및 푸시
git add .
git commit -m "Initial deployment"
git push origin main
```

### 5단계: GitHub Pages 활성화
1. 저장소 Settings → Pages
2. Source: `Deploy from a branch`
3. Branch: `main` / `/ (root)`
4. Save

### 6단계: 접속 확인
- URL: `https://사용자명.github.io/corporate-strategy/`
- 자동으로 `대표님_Intro_202512_공감.html`로 이동

## 🔧 문제 해결

### 파일이 열리지 않을 때
- GitHub Pages가 활성화되었는지 확인
- 파일 경로가 올바른지 확인
- 브라우저 캐시 삭제 후 재시도

### 이미지가 보이지 않을 때
- 파일명에 공백이 있는지 확인
- 대소문자 구분 확인
- 파일이 실제로 업로드되었는지 확인

### 링크가 작동하지 않을 때
- 모든 파일이 같은 폴더에 있는지 확인
- 파일명이 정확한지 확인 (특수문자, 공백 등)
